# P-ADO Analysis and Plotting

## Purpose

The `analysis` directory contains helper modules and notebooks for inspecting P-ADO CSV outputs, producing two-dimensional density plots, projecting one-dimensional distributions, and diagnosing singular or near-singular Jacobian regions.

## Supported environment

Python 3.8 or later is required. Ubuntu 20.04 or later is recommended. On Ubuntu 18.04, please install Python 3.8 explicitly; the default Python 3.6 environment is not supported.

## Required input files

The analysis workflow expects CSV files produced by the calculation package. No example CSV files are included in the repository.

## Simplest analysis setup

The analysis scripts do not require installing P-ADO as a package if they are used from the repository directory. Install the analysis dependencies first:

```bash
python -m pip install -r analysis/requirements_analysis.txt
```

Then open the notebooks or edit the path in the scripts/notebooks to point to your generated CSV or CSV.GZ file, for example:

```python
CSV_PATH = Path(r"path\to\file_name.csv.gz")
```

Use ordinary analysis with `*_all.csv` or `*_all.csv.gz` files. Use singular diagnostics with `*_detJ_singular.csv` or `*_detJ_singular.csv.gz` files. Do not assume that `*_detJ_regular.csv` or `*_detJ_regular.csv.gz` files form a complete rectangular grid.

## Ordinary analysis workflow for `*_all.csv` or `*_all.csv.gz`

Use `YAnalysis.py`, `YPlot.py`, and `YExtract.ipynb` for ordinary analysis of complete all-data files:

```text
*_all.csv
*_all.csv.gz
```

These files retain the full delta-sigma/I grid and are suitable for the grid-based density and projection helpers.

## Singular diagnostic workflow for `*_detJ_singular.csv` or `*_detJ_singular.csv.gz`

Use `YAnalysis_singular.py`, `YPlot_singular.py`, and `YExtract_singular.ipynb` for singular or near-singular Jacobian diagnostics:

```text
*_detJ_singular.csv
*_detJ_singular.csv.gz
```

The singular workflow uses scatter plots and binned histograms that do not assume a complete rectangular grid.

## Notebook usage

Open the notebooks from the `analysis` directory or configure the working directory so local imports such as `from YAnalysis import ...` and `from YPlot import ...` resolve correctly. Install dependencies with:

```bash
python -m pip install -r analysis/requirements_analysis.txt
```

## How to replace file paths

Replace the placeholder path in each notebook with a CSV generated by your own P-ADO run, for example:

```python
CSV_PATH = Path(r"path\to\file_name.csv.gz")
```

For singular diagnostics, use:

```python
CSV_PATH = Path(r"path\to\file_name_detJ_singular.csv.gz")
```

Do not leave local machine-specific absolute paths in notebooks intended for release.

## Warning about regular files

The regular file is generated after removing singular or near-singular Jacobian points. Therefore, the delta-sigma/I and AT-sigma/I points may no longer form a complete rectangular grid. Grid-based plotting routines may not be directly applicable to regular files unless the missing points are handled explicitly.

## Plotting notes

Curvilinear P-ADO observable-space densities should be plotted with scatter-style helpers rather than rectangular `pcolormesh` routines, because folded observable-space points can be connected incorrectly by grid plotting. Singular diagnostic plots use sampling and histogram binning controls to keep large outputs manageable.
